🎨 Implement This Manually in Canva

A no-code path to the same look: instead of the generator engine in explainer.html, build the background and the lower-third labels as images, then place, time, and animate them by hand in Canva. Useful if you want to hand-tweak the design directly, or you're handing this off to someone who isn't going to touch code.

🎬 Demo

Example Canva design: canva.link/nqqjs8gf15aaj5r — open it, duplicate it into your own account, and use it as the starting template for the steps below.

🪜 Steps

1. Create the background image in Gemini

Use the same content-matched prompt this project uses for explainer.html — the scene layout should mirror the actual flow your lower thirds describe (e.g. cluster → gateway → pod), not just be generically on-theme. The exact prompt that produced this project's background is documented on explainer.html (🖼️ Background image credit panel) and in MASTER_SPEC.md §10 — copy it as a starting point and adjust for your own content. Generate at 1920×1080 (16:9), and leave the bottom ~20–25% of the frame darker/clearer — that's where your labels will sit. Upload the result as your Canva design's background.

2. Create the lower thirds in Gemini

Generate each label as its own image (icon + text on a rounded pill), matching this project's visual spec from problem.md: rounded rectangle, 24–30px corner radius, background #0B1B1E at 85% opacity, a cyan (#00D2D2) icon on the left, white serif text on the right. Generate one image per label (transparent background) so you can place and time each one independently in Canva — don't bake all three into a single image.

3. Place the lower thirds in the correct places

Don't eyeball it — explainer.html's 🏗️ Builder section computes exact x/y pixel coordinates (at the background image's native 1920×1080) for wherever your text/emoji fields currently are, and lets you drag X/Y sliders to preview custom placement before you commit to it. Match those numbers to Canva's position fields (select an element → the Position panel shows X/Y) for pixel-accurate placement instead of guessing.

4. Set the timing for the lower thirds

This project's default three-stage timing is 0ms, 3200ms, and 6400ms from playback start (each card settles, then holds ~2.2s before the next one rises — see MASTER_SPEC.md §3.1). In Canva, select each label on the timeline and set its entrance to start at the matching second (0s, 3.2s, 6.4s) so the pacing matches.

5. Set up the animations on enter (rise animation)

Select each label → Animate → choose Rise from Canva's entrance animations. This is the closest built-in match to the site's actual motion (a damped-spring rise, per problem.md: amplitude 120px, ζ=0.5, ωn=11) — Canva's Rise won't reproduce the exact overshoot/bounce physics, but it gets the same "springs up from below and settles" read. Apply it to each of the three labels individually so they trigger at the times set in step 4, not all at once.

🤔 When to use this vs. the generator

  • Use this Canva path if you want to hand-edit the design directly, need it inside an existing Canva project/brand kit, or are handing off to someone non-technical.
  • Use explainer.html/video-recorded.html/video-live.html if you want the exact coded spring physics, want to type new text and get an instant result, or need a downloadable video/live overlay — see mcp-guide.html for driving those programmatically too.
