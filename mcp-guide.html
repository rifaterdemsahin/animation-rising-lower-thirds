📖 How to Use This Project Over an API

Three ways to drive this toolbox without clicking through the UI, from "no new code" to "a hosted HTTP API you can call from anywhere." Full detail lives in MCP_GUIDE.md — this page mirrors it on-site.

🖱️ Option 1 — browser automation, no new code

If your MCP client already has browser tools (e.g. claude-in-chrome's navigate / computer / javascript_tool), you already have everything needed:

  1. navigate to video-recorded.html?icon1=💼&text1=Real+Jobs&icon2=👤&text2=Real+People&autorecord=1.
  2. Poll for document.getElementById("status").textContent === "Recording ready.".
  3. Read document.getElementById("download-link").href (a blob: URL).

🧩 Option 2 — build your own MCP server

Wrap a headless browser (Node + Puppeteer, or any language) around the same URL API. Full sketch in MCP_GUIDE.md. Useful if you're already building an MCP server and want render_lower_third as one of its tools.

🚀 Option 3 — the hosted API (live now)

Option 2's approach, already built in Go and deployed to Fly.io — no server to stand up yourself. Source: server/main.go (server/ in this repo). It drives headless Chrome (chromedp) against video-recorded.html?...&autorecord=1 — the exact same engine and URL API as Options 1 and 2, just hosted.

Base URL: https://lower-thirds-api.fly.dev

  • GET /healthz — no auth, returns 200 ok.
  • POST /render — requires Authorization: Bearer $API_TOKEN; JSON body {icon1, text1, icon2, text2}; response body is video/webm bytes. 
curl -X POST https://lower-thirds-api.fly.dev/render \
  -H "Authorization: Bearer $API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"icon1":"💼","text1":"Real Jobs","icon2":"👤","text2":"Real People"}' \
  -o lower-thirds.webm

$API_TOKEN is a Fly.io secret, never committed — see server/README.md for how to get or rotate it.

🤔 Which to use

  • Prototyping, one-off generation, or you already have a browser-automation MCP server → Option 1.
  • Building your own MCP server / embedding this in another toolchain → Option 2.
  • Just need a video back from an HTTP call, right now → Option 3.
