📖 How to Use This Project Over an API

Three ways to drive this toolbox without clicking through the UI, from "no new code" to "a hosted HTTP API you can call from anywhere." Full detail lives in MCP_GUIDE.md — this page mirrors it on-site.

🖱️ Option 1 — browser automation, no new code

If your MCP client already has browser tools (e.g. claude-in-chrome's navigate / computer / javascript_tool), you already have everything needed:

  1. navigate to video-recorded.html?icon1=🖥️&text1=Kubernetes+Test+Cluster&icon2=🚪&text2=Ingress+Gateway&icon3=📦&text3=Pod&autorecord=1 (the page's own default content — omit the params entirely for the same result).
  2. Poll for document.getElementById("status").textContent === "Recording ready.".
  3. Read document.getElementById("download-link").href (a blob: URL).

🧩 Option 2 — build your own MCP server

Wrap a headless browser (Node + Puppeteer, or any language) around the same URL API. Full sketch in MCP_GUIDE.md. Useful if you're already building an MCP server and want render_lower_third as one of its tools.

🚀 Option 3 — the hosted API (live now)

Option 2's approach, already built in Go and deployed to Fly.io — no server to stand up yourself. Source: server/main.go (server/ in this repo). It drives headless Chrome (chromedp) against video-recorded.html?...&autorecord=1 — the exact same engine and URL API as Options 1 and 2, just hosted.

Base URL: https://lower-thirds-api.fly.dev

  • GET /healthz — no auth, returns 200 ok.
  • POST /render — requires Authorization: Bearer $API_TOKEN; JSON body {icon1, text1, icon2, text2} (2-card only for now — see §"Which to use"); response body is video/webm bytes. 
curl -X POST https://lower-thirds-api.fly.dev/render \
  -H "Authorization: Bearer $API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"icon1":"🖥️","text1":"Kubernetes Test Cluster","icon2":"🚪","text2":"Ingress Gateway"}' \
  -o lower-thirds.webm

$API_TOKEN is a Fly.io secret, never committed — see server/README.md for how to get or rotate it.

🗣️ Prompts to give an AI agent

If you're asking Claude (or any AI agent) to actually consume this project rather than reading about it, hit 📋 Copy on any block below and paste it straight into your agent's chat — each one is a complete, self-contained instruction it can act on to trigger one of the outputs above and hand you back the result.

🖱️ For Option 1 (an agent with browser tools)

Using your browser tools, navigate to
https://rifaterdemsahin.github.io/animation-rising-lower-thirds/video-recorded.html?autorecord=1
(the page defaults to the Kubernetes Test Cluster / Ingress Gateway / Pod
example — add &icon1=&text1=&icon2=&text2=&icon3=&text3= to override it).
Wait until the on-page status reads "Recording ready.", then read the
download-link element's href and give me the video.

🧭 For the Kubernetes explainer (Option 1, staged content)

Using your browser tools, navigate to
https://rifaterdemsahin.github.io/animation-rising-lower-thirds/explainer.html?autoplay=1
and describe what happens on screen as each of the three stages rises
(Kubernetes Test Cluster, then Ingress Gateway, then Pod). Add
&text1=&emoji1=&text2=&emoji2=&text3=&emoji3= to swap in your own three
stages instead.

🧩 For Option 2 (an agent building its own MCP server)

Build me an MCP server with a render_lower_third(icon1, text1, icon2, text2)
tool that returns a video/webm file. Drive it by opening
https://rifaterdemsahin.github.io/animation-rising-lower-thirds/video-recorded.html
with those params + &autorecord=1 in a headless browser, waiting for
"Recording ready.", then fetching the download-link blob as base64.
See MCP_GUIDE.md "Option 2" in the animation-rising-lower-thirds repo
for a working Puppeteer sketch.

🚀 For Option 3 (an agent with HTTP/curl access — no browser needed)

Call this API to render a lower-thirds video, then save the result:

POST https://lower-thirds-api.fly.dev/render
Headers: Authorization: Bearer $API_TOKEN, Content-Type: application/json
Body: {"icon1":"🖥️","text1":"Kubernetes Test Cluster","icon2":"🚪","text2":"Ingress Gateway"}

The response body is the raw video/webm file — write it to lower-thirds.webm.
($API_TOKEN is a secret; ask the repo owner or see server/README.md. This
endpoint currently supports 2 cards only, not the full 3-stage example.)

🤔 Which to use

  • Prototyping, one-off generation, or you already have a browser-automation MCP server → Option 1.
  • Building your own MCP server / embedding this in another toolchain → Option 2.
  • Just need a video back from an HTTP call, right now → Option 3.
