🎨 Why Manual Canva Still Makes Sense

cost-analysis.html is honest that the manual Canva path is the slowest and most expensive of the options on this site — ~15–30 minutes and ~$0.30+ in generation, versus seconds and cents for the coded tools. That's a real cost. It's also not the whole story: raw minutes-per-project isn't the only thing that matters, and most of that cost is a one-time setup tax, not a recurring one — this page is about both halves.

⚖️ What the time/cost numbers don't capture

cost-analysis.html's "time to use" column measures one thing: how long it takes to produce an asset. It doesn't measure who can produce it, or what happens after — and that's where Canva earns back the time it costs upfront:

  • No code, no dev environment. Anyone on a team can open a Canva design and change text, retime an entrance, or swap a color — no git, no browser dev tools, no understanding of springState() required. The coded tools' Builder is genuinely easy too, but it's still a webpage someone has to be shown; Canva is a tool most non-engineers already know.
  • Real-time collaboration. Multiple people can be in the same Canva file at once — a designer placing labels while someone else writes copy. Nothing on this site has a multi-user equivalent; the Builder is a single-browser-tab tool.
  • Survives this project going away. A Canva design doesn't depend on lower-thirds-api.fly.dev staying up, GitHub Pages staying free, or this repo being maintained. It's a self-contained asset in your own Canva account.
  • No fixed spec. The coded engine is deliberately locked to problem.md's exact visual spec — that's a feature for consistency, but a constraint if you want something the spec doesn't allow (a fourth card, a different shape, mixing in stock photography). Canva has no such ceiling.

✂️ How to cut the time cost that's left

The ~15–30 minute estimate in cost-analysis.html assumes starting from nothing every time. Almost none of these steps actually need repeating per project:

  1. Duplicate the template instead of starting fresh. The biggest lever by far. The example design linked from canva-guide.html already has the background, three labels, placement, timing, and Rise animations set up — duplicating it and swapping text/timing turns a 20-minute build into a 2–3 minute edit.
  2. Build a small image library once, reuse it everywhere. Generate a handful of background styles and common label icon/text pairs up front and keep them in a Canva folder or your Brand Kit. Per cost-analysis.html, reusing an already-generated image costs $0 — the ~$0.30+ figure only applies when every image is new.
  3. Copy exact numbers instead of eyeballing them. explainer.html's 🏗️ Builder computes precise x/y positions and the 0ms/3200ms/6400ms stage timing for you — paste those straight into Canva's Position panel and timeline instead of guessing and iterating.
  4. Save the animated label as a reusable element. Once one label has its Rise animation configured correctly, copy-pasting it (then just changing its text/icon and re-timing) is faster than configuring the animation from scratch each time.
  5. Split the work. Because Canva supports real-time multi-user editing, image generation, placement, and copywriting can happen in parallel across different people — something a single-tab coded workflow can't do at all.

📉 First use vs. every use after

First project (from scratch)Every project after (duplicated template)
Time ~15–30 minutes ~2–5 minutes — duplicate, swap text, adjust one image if the content genuinely changed
Cost ~$0.30–0.35 (4 new images) $0 if reusing the background and label styles; ~$0.08 if only one new label needs a new image

That "every project after" column is close to — sometimes faster than — the coded tools once you count the time to actually type content into a form and review the result. The gap cost-analysis.html shows is mostly a first-time setup cost, not a per-project one.

🤔 Rule of thumb

  • Pick Canva if non-engineers will need to make ongoing edits, if you want the design to outlive this project, or if you'll want to deviate from the fixed visual spec — the upfront setup cost pays for itself on the second use.
  • Pick the coded tools (video-recorded.html, video-live.html, explainer.html) for a one-off you won't touch again, or when you need the exact spring physics — Canva's Rise animation approximates it but doesn't reproduce the damped-harmonic math exactly.
